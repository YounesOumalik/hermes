### 3.3 ⚠️ Problèmes de code détectés

#### 🔴 Bug résiduel : fallback `eaumalik` dans `login/page.tsx`

```tsx
// Lignes 25 et 30 — STILL PRESENT IN SOURCE
const backendApiUrl = process.env.NEXT_PUBLIC_API_URL || "https://agentai.eaumalik.com/api";
```

Bien que le build actuel utilise `.env.production` (donc fonctionne), le code source contient encore le fallback. **Si quelqu'un rebuild sans `.env.production`, le bug revient.**

#### 🟡 Incohérence : 2 patterns d'URL différents

- `login/page.tsx` → utilise `backendApiUrl` (URL absolue construite depuis env)
- `page.tsx`, `admin/page.tsx` → utilisent **URLs relatives** `/api/auth/me` ✅

Les URLs relatives sont **meilleures** car elles héritent du domaine courant. Il faudrait uniformiser.

#### 🟡 Pas de gestion d'erreur 401 globale

Chaque page refait `localStorage.getItem("access_token")` puis `fetch` puis `if !res.ok redirect to /login`. Pas de helper centralisé. Risque de duplication.

#### 🟡 `page.tsx` fait 761 lignes dans un seul composant

Le dashboard contient workspace + agents + chat + messages + modals. Devrait être découpé en composants (`<Sidebar>`, `<ChatPanel>`, `<AgentModal>`, etc.).
